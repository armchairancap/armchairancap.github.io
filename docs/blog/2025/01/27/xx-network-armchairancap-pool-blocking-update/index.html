Update on pool-blocking progress

Introduction​

As described here, my pool-blocking pool is up and running!

This post provides some updates for community members and my nominators.

High-level Overview​

Commissions​

My goal is to cover my hosting expenses and get some minimal rewards for the work I do for the community (meaning, not just "OS maintenance", but development, testing and implementing these blocking approaches).

As the two nodes are new, I had no idea how they may perform.

I started with 5%, but cut it a bit to keep it roughly acceptable to nominators.

info

Each node currently makes less than 10 xx per (0.05) per day.

It doesn't cover my hosting, but I'll think about that at a later time once pool blocking is in full swing.

Blocking​

I've been developing and testing aproaches for blocking, reporting, etc.

This afternoon I started blocking "in production". The first target is MONEYTEAM: they run more than 5 nodes, and do it in different regions. An enemy of the network.

First, I find their cMix IDs in the wallet.

Then I use the workflow from my Github repository (see Appendix A) to find their IP addresses, add them to a list and then run the script that denies them access. The script - as posted on Github - works fine with generic ufw settings recommended in the xx Network Handbook, so you may use it by yourself if you want.

Monitoring and reporting​

This is time consuming. I've been planning, evaluating and developing an approach to gather data and produce reports.

This is important for several reasons, for example:

  • Nominators who potentially compromise a few percent in yield may want to know if I'm really blocking as I say I do.
  • I need to know if I"m realy blocking (as I can't claim I am if I'm not) and how well.
  • I need the data to see how effective my approach is, and to estimate losses caused to the pools.

So far I got monitoring working in a prototype. That means it's not "automated", but the steps work.

Today I deployed the rules to ARMAIRANCAP-ALFA targeting MONEYTEAM as I've mentioned earlier.

I have a report generator that lists cMix rounds failed due to timeouts in last 24 hours.

info

That's actually from yesterday - an accidental timeout (before I started blocking) - but I can't tell which round failed to my blocking and which to other reasons.

My report looks for specific (pool cMix IDs stored in a DB), so this doesn't list rounds that failed with random nodes.

The second table shows total damage to the pool (see Appendix B). You can see there's a lot more failed rounds today (Jan 27) after 4am UTC and that eXRzev8PC/OXj669SLGwy2zQmr6p8uCmPy0oF03AfiwC got hit the hardest, 4 failed cMix rounds.

Let's see that first round, 317854252, in the Dashboard.

If we pick the last round from the first table - 318132140 - we can see that's cMix ID itrNbIlTQwJfD4SKB6dCQM/qJOpblYZzXgMp10QGKWgC which is a wallet with on-chain ID moneyteam_06 running out of the UK (see Appendix B).

This is just over a 7 hour period and from only one of my nodes!

I hope this is as motivating to my nominators as it is to me!

Analysis​

Based on this small sample, let us assume 170 points gets taken away from them every eight hours. That's close to 500 per day, and 1,000 per day if both ALFA and CHARLIE be blocking.

note

I plan to add additional nodes. Sorry, but I'm not sorry!

If I get to 2,000 points per day, with this I could deplete MONEYTEAM's earnings by 60,0000 per month. That should dent their earnings (and mine, but that's fine) by 3-4%.

In theory nothing prevents me from running 6 or 8 nodes, but for that I'd have to not lose money running them.

That's why I need analytics, to help me scale to as many nodes as possible, while still being able to cover expenses and attract nominators.

warning

I can start blocking CRYPTOCALIBUR today, but there's no way to earn 10% for validators if I'm blocking 30 nodes.

So I can't "really" start blocking them now - I'd lose nominators, and then my nodes wouldn't get elected, and then everything would be in vain.

The good news here is they can't beat me. We both lose cMix points, but I don't have to make a profit for myself and - as pool operators - they do.

Next steps​

This week I'll start blocking MONEYTEAM from CHARLIE as well.

I need to do more work building out monitoring, observability and reporting.

info

As one of many examples, that's obvious here - see how in Appendix B I have on-chain ID and pre-comp failure rate in a text file. That should be pulled dynamically from on-chain and dashboard, but it all takes time...

I don't even want to automate all that, but just copying cMix IDs, wallet IDs and on-chain IDs, and keeping it up to date takes a lot of time.

Once I reach the minimum level of monitoring,reporting and automation, I may add additional validator nodes and start blocking MONEYTEAM from those nodes as well.

Finally, I plan to provide a Web site where my nominators and other friends of xx Network can view these statistics in near real-time. That

note

As an aside: I've spent a lot of time on this already, and it's sad that I spend time on this when I could be building apps or doing something more useful.

But the community has a socialist scum problem that has to be dealt with as a matter of urgency.
