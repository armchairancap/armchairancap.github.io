Connect to xx Network

The Speakeasy Web application downloads and runs in your browser, from which it connects to community-operated xx Network gateway nodes.

info

Speakeasy's end-to-end encryption happens in the browser, and metadata protection on xx Network.

Choices​

Depending on several factors, many combinations are possible for self-managed server at home or in the cloud, and 3rd party Speakeasy Web server.

This is not an exhaustive list:

info

One can't run own xx Network gateway server or connect to one particular gateway: that's where trustless metadata stripping and shredding happens and requires a sophisticated network.

Access Speakeasy from your browser​

Approaches in terms of privacy and security:

  • Hosted by trusted third party
    • Regular privacy: visit the official xx Network-hosted instance of Speakeasy. This option exposes your client IP to the operator (The xx Foundation).
    • High privacy: use a Socks5 proxy to access Speakeasy Web app and xx Network over Tor network or VPN. This option hides your client IP from the server operator, but also your client IP from community-operated xx Network gateways.
  • Self-hosted server (at home or in the cloud)
    • Enhanced privacy: deploy your own instance of Speakeasy Web app. Your client IP is known to community-operated xx Network gateways, but everything else is encrypted.
    • Top privacy: host own instance and access xx Network gateway nodes via Tor Network or VPN. This doesn't identify you to Speakeasy Web site owner or 3rd parties as a Speakeasy user, and also hides your client IP from community-operated xx Network gateways.

Trusted Speakeasy Web server​

The main idea here is to avoid using Speakeasy Web site on a compromised or malicious server.

"Trusted" is a relative term but, assuming the official Speakeasy site is trusted, you may use speakeasy.tech.

Additional examples of a trusted Speakeasy Web server would be a Speakeasy Web server running at friends home.

Self-managed Speakeasy instance​

  • Run Speakeasy on own Linux server or VM, cloud-based or own

    • Use Speakeasy container on Docker or Podman, or
    • Install Speakeasy Web app on Node.js

  • Access Speakeasy over Tor network using Socks5 proxy

    • Tor Socks5 proxy
    • Arti Socks5 proxy
tip

Speakeasy administrator's guide has information related to deploying and operating own Speakeasy Web server.

If your environment doesn't have a shared Tor or Arti Socks proxy, you may be able to run your own.

To use a Tor Socks5 proxy, install Tor Browser. Then enable and start Tor service.

For Arti, download it, build, and run.

arti proxy

Use your regular browser to connect to Tor or Arti Socks proxy using something like socks5://localhost:9150 for localhost-based proxy, or socks5://proxy.local:9150 for LAN instances.

tip

If your Speakeasy Web server runs locally or can be trusted, bypassing Socks proxy for that host may load Speakeasy faster.
